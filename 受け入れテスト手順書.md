# 受け入れテスト手順書(Electron配布版)

Electron のインストーラ版とポータブル版のクリーン環境受け入れテスト。1回のテストセッションで両方を検証する。所要目安: 1〜2時間。

## 0. テスト対象物(作者PC側で準備)

| 配布物 | 場所 |
|---|---|
| インストーラ | `release\electron\ChapterFlow Setup 0.1.0-beta.5.exe` |
| ポータブル exe | `release\electron\ChapterFlow 0.1.0-beta.5.exe` |
| ライセンス | `release\electron\LICENSE` |
| 利用ガイド | `release\electron\利用ガイド.md` |
| SHA-256 | `release\electron\SHA256SUMS.txt` |

上記5点を USB メモリまたはクラウド経由でテスト環境へ持ち込む。**`win-unpacked/` フォルダは持ち込まない**(配布物ではない)。

配布前にPowerShellで `Get-FileHash -Algorithm SHA256 <exeのパス>` を実行し、
`SHA256SUMS.txt` の値と一致することを確認する。

テスト用に使い捨ての Gemini か DeepSeek の APIキーを1つ用意しておく(生成テストに必要。テスト後に無効化してよい)。

## 1. テスト環境の用意

優先順:

1. **別の Windows PC(推奨)** — 開発ツールが入っていないもの
2. **同じPCの新規ローカルアカウント** — 設定 → アカウント → 他のユーザー → 追加(Microsoft アカウントなしのローカルアカウント)
3. **Windows Sandbox / VM** — Windows Pro なら「Windows サンドボックス」が手軽(ただし再起動で消えるので1セッションで完了させる)

`Documents\ChapterFlow` と `%APPDATA%\ChapterFlow` が存在しないこと(存在したら削除してから開始)。旧名互換テストを行わない場合は `Documents\Yumeweaving` と `%APPDATA%\Yumeweaving` も存在しない状態にする。

## 2. フォルダの使い方

| 用途 | 場所 | 備考 |
|---|---|---|
| ポータブル exe の置き場所 | `デスクトップ\chapterflow-portable\` などのローカルフォルダ | インストール版と混ざらないように分ける |
| 作品データ(自動生成) | `%USERPROFILE%\Documents\ChapterFlow` | インストーラ版・ポータブル版で共通。APIキー(config\credentials.json)と Electron のポート記録もここ |
| Electron のウィンドウ状態 | `%APPDATA%\ChapterFlow\window-state.json` | インストーラ版・ポータブル版で共有される |

**注意**: 2形態とも同じ `Documents\ChapterFlow` を使う。テスト間は必ず前のアプリを終了してから次へ進む。

| A12 | × ボタンで終了した場合も A11 と同じ確認 | 同上 | |

| A15 | アンインストール(設定 → アプリ) | `Documents\ChapterFlow` が**消えない**。作品データが無事 | |

## 5. パートC: 旧名からの手動移行確認

旧Yumeweavingを使ったことがある環境、または旧フォルダを再現した環境で別途実施する。

| # | 手順 | 期待結果 | ✓ |
|---|---|---|---|
| C1 | `Documents\ChapterFlow` を作らず、作品・configを含む `Documents\Yumeweaving` だけを置いて起動 | `Documents\ChapterFlow` が既定の保存先になり、旧フォルダは自動選択されない | |
| C2 | `%APPDATA%\ChapterFlow` を作らず、旧 `%APPDATA%\Yumeweaving` だけを置いて起動 | 旧ウィンドウ状態、テーマ、作成途中セッションが引き継がれる | |
| C3 | 作品を含む `Documents\ChapterFlow` と `Documents\Yumeweaving` の両方を置いて起動 | 新しい `ChapterFlow` 側が優先される | |
| C4 | 空の `Documents\ChapterFlow` と、作品を含む `Documents\Yumeweaving` を置いて起動 | `ChapterFlow` 側が使われ、旧作品は自動表示されない | |
| C5 | 旧作品をバックアップし、内容を `Documents\ChapterFlow` へコピーしてからChapterFlowを起動 | コピーした旧作品と設定が表示される | |

## 6. 後片付け

1. アプリのアンインストール(パートAで実施済みならスキップ)とポータブル exe の削除
2. `Documents\ChapterFlow` と `%APPDATA%\ChapterFlow` の削除
3. テスト用 APIキーの無効化(各社のコンソールから)
4. テスト用ローカルアカウントを使った場合はアカウント削除
