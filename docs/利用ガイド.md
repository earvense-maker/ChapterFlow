# Yumeweaving 利用ガイド

## Yumeweaving とは

Yumeweaving は、自分専用の連載小説をローカルPC上で育てるWebアプリです。
作品設定、これまでの流れ、種メモをもとに、LLM APIを使って次の話を生成できます。
データはPC内に保存され、スマホ共有も同じWi-Fi内でだけ使う想定です。

## 必要なもの

- Windows 10 または Windows 11
- Node.js 20 以上
- Gemini、DeepSeek、OpenAI のいずれかの API キー

Node.js は公式サイトの [Download Node.js](https://nodejs.org/en/download) から Windows Installer を入手してインストールしてください。公式サイトでは現在の LTS 版が案内されます。インストール後、コマンドプロンプトで `node -v` が表示されれば準備完了です。

## インストールと起動

1. 配布された zip を、デスクトップや `Documents` などのローカルフォルダに展開します。
2. 共有フォルダやネットワークドライブ上で直接起動せず、必ずローカルに展開してから使ってください。
3. 展開したフォルダ内の `Start-Yumeweaving.bat` をダブルクリックします。
4. 初回起動時は依存パッケージの取得で数分かかることがあります。
5. ブラウザ風のウィンドウが開いたら、作品を作成できます。

起動用の黒いウィンドウはサーバー本体です。Yumeweaving を終了するときは、そのウィンドウを閉じてください。

## API キーの取得と登録

いずれか1つ以上の API キーを用意してください。

- Gemini: [Google AI Studio API keys](https://aistudio.google.com/app/apikey) に Google アカウントでログインし、API キーを作成します。
- DeepSeek: [DeepSeek API Keys](https://platform.deepseek.com/api_keys) にログインし、API キーを作成します。
- OpenAI: [OpenAI API Keys](https://platform.openai.com/api-keys) にログインし、API キーを作成します。

作成した API キーは、アプリ内の「技術設定」タブから登録します。

**生成はあなた自身のAPIキーで行われ、利用量に応じて各社から課金されます。利用前に各社の料金ページと上限設定を確認してください。**

## データの保存場所とバックアップ

作品データと設定は `Documents\Yumeweaving` に保存されます。バックアップするときは、このフォルダを外付けドライブやクラウドストレージへコピーしてください。

API キーは `Documents\Yumeweaving\config\credentials.json` に平文で保存されます。バックアップや相談のためにフォルダを人へ渡す場合は、`config` フォルダを除いてください。

配布フォルダを削除しても、`Documents\Yumeweaving` を削除しない限り作品データは残ります。

## スマホから使う（任意）

同じ Wi-Fi に接続しているスマホから使う場合は、PC側で `Start-Yumeweaving-LAN.bat` をダブルクリックします。

起動ログに `http://<PCのLAN IP>:3001/?token=...` というURLが表示されます。スマホのブラウザでそのURLを開くと、以後はトークンなしでも同じブラウザから利用できます。

初回に Windows ファイアウォールの確認が出たら、プライベートネットワークだけ許可してください。カフェや駅などの公共Wi-Fiでは使わず、自宅など信頼できるWi-Fiでのみ使ってください。

トークン付きURLを知っている人は、同じWi-Fi内から作品の閲覧・編集・生成を実行できます。SNSやチャットに貼らないでください。

## トラブルシューティング

### Node.js が見つからない

`Node.js 20+ をインストールしてください` と表示された場合は、Node.js をインストールしてからもう一度 bat を起動してください。インストール後に反映されない場合は、PCを再起動してください。

### ポート3001が使用中と表示される

既に Yumeweaving が起動しているか、別のアプリが3001番ポートを使っています。まず開いている Yumeweaving の黒いウィンドウを閉じてください。

解決しない場合は PowerShell で次を確認できます。

```powershell
Get-NetTCPConnection -LocalPort 3001
```

必要な場合だけ、次のコマンドで3001番ポートを使っているプロセスを終了できます。

```powershell
Stop-Process -Id (Get-NetTCPConnection -LocalPort 3001).OwningProcess -Force
```

### Cannot find module と表示される

初回起動中に黒いウィンドウを閉じるなどして依存パッケージの取得が中断されると、不完全な `node_modules` フォルダが残ることがあります。配布フォルダ内の `node_modules` フォルダを削除してから、もう一度 `Start-Yumeweaving.bat` または `Start-Yumeweaving-LAN.bat` を起動してください。

### スマホから開けない

PCとスマホが同じWi-Fiにいるか確認してください。Windows ファイアウォールで拒否してしまった場合は、Windows の「ファイアウォールによるアプリケーションの許可」から Node.js をプライベートネットワークで許可し直してください。

スマホでは、必ず起動ログに表示されたトークン付きURLを最初に開いてください。トークンなしで直接アクセスすると認証画面または401エラーになります。

### 起動中のログを確認したい

`Start-Yumeweaving.bat` または `Start-Yumeweaving-LAN.bat` を起動した黒いウィンドウにログが表示されます。エラーが出た場合は、その画面の内容を確認してください。
