# プリセット再設計 設計書

作成日: 2026-07-19
対象: 作品作成時に選択するプリセット(現行10カテゴリ)を、新7カテゴリに全面入れ替えする。

## 1. 目的と設計原則

現行プリセットには次の問題がある。

- 会話量を制御する軸が4カテゴリ(文体/描写密度/会話量/展開テンポ)に分散し、矛盾する組合せが選べる。
- 「視点スタイル」と「語りの距離感」が重複し、こちらも矛盾が選べる。
- text が「〜な物語。」形式の形容で、LLM への指示として効果が薄い。
- 「禁止事項」が単一選択ラジオ(複数選べない・解除できない)。
- 濡れ場のデフォルトが `suggestive` で、全新規作品のプロンプトに性的場面の指示が入る。

新設計の原則:

1. **1つの軸は1カテゴリだけが持つ**(直交性)。
2. **選ばなければプロンプトに何も入らない**。「指定しない」が原則デフォルト。
3. **text は行動指示(「〜する/〜しない」)で書く**。形容詞で書かない。
4. **既存の強い機構と競合するカテゴリを持たない**。文体の細部は文体見本(styleSample)、禁止事項は NG メモ(negative memory)に一本化する。

## 3. データモデル変更

### 3.1 `src/shared/types.ts` — ActivePresets

```ts
export interface ActivePresets {
  narration: string;            // 必須。デフォルト 'third-close'
  aftertaste?: string[];        // 最大2件
  emotionDisplay?: string;
  sceneProgression?: string;
  chapterEnding?: string;
  painLevel?: string;
  intimacy?: string;
}
```

旧フィールド(genre/style/pov/distance/pacing/density/conversation/relationshipPacing/constraint)は削除。

### 3.2 `src/shared/defaults.ts`

```ts
export const DEFAULT_ACTIVE_PRESET_IDS = {
  narration: 'third-close',
} satisfies ActivePresets;
```

### 3.3 `PresetsFile`(src/shared/types.ts)のミラーフィールド廃止

現行は `genrePreset` 〜 `intimacyPreset` の10フィールドが `project.activePresetIds` のミラーとして presets ファイルに二重保存されている(projectService.ts の createProject / updateActivePresets が書き込み)。

- **選択状態の正本は `project.activePresetIds` に一本化し、ミラーの書き込みを全て廃止する。**
- `PresetsFile` の旧 `*Preset` フィールドは型から削除する(読み込み時に余分なキーがあっても無害なので、読み捨てる)。
- `PresetsFile` に残すのは `userCustomPromptParts` / `baseSystemPrompt` / `customSystemPrompt` のみ。
- クライアント(WorkSettingsTab)は `presets.intimacyPreset ?? project.activePresetIds.intimacy` のようなフォールバック参照をやめ、`project.activePresetIds` のみ参照する。

### 3.4 バリデーション

`projectService.ts` の updateProject バリデーション(現在「各値は string であること」)を変更:

- `aftertaste` は `string[]`(各要素 string、最大2件)を許可。
- その他のキーは string。
- 未知のカテゴリキーは拒否ではなく **無視(読み捨て)** とする(後方互換のため)。

## 4. 旧データの移行(読み込み時正規化)

一括変換バッチは作らない。`normalizeProjectType`(defaults.ts)と同じ方式で、**API/読み込み境界での正規化**として実装する。

### 4.1 新規モジュール `src/shared/presetMigration.ts`

```ts
export function normalizeActivePresetIds(raw: unknown): ActivePresets
```

- `raw` に `narration` キーがあれば新形式とみなし、既知 id の検証だけ行う(不正 id はキーごと落とし、narration 不正時は `third-close`)。
- なければ旧形式とみなし、下記マッピングを適用する。

### 4.2 旧 id → 新 id マッピング表

適用順は表の上から。同じ新キーへの書き込みが競合した場合は**先勝ち**(上書きしない)。`aftertaste` のみ配列に追加(最大2)。表に無い旧 id はすべて**破棄**(新設定に反映しない)。

| pov.third-person-fixed | narration.third-close |
| pov.third-person-close | narration.third-close |
| pov.per-scene | narration.third-close |
| intimacy.(none 以外) | intimacy.(同一 id) |
| intimacy.none | (破棄=未設定) |
| distance.emotional | emotionDisplay.expressive |
| distance.factual | emotionDisplay.restrained |
| style.quiet | emotionDisplay.restrained |
| style.afterglow | chapterEnding.lingering |
| style.tense | aftertaste に searing を追加 |
| pacing.slow | sceneProgression.immersive |
| pacing.fast | sceneProgression.brisk |
| pacing.action-driven | sceneProgression.brisk |

- narration が決まらなかった場合は `third-close` を入れる(必須のため)。
- 正規化はプロジェクト読み込み境界(projectService の read/一覧、setupCommit、updateProject のマージ前)で適用する。**ディスクへの書き戻しは次回保存時で構わない**(読むたび正規化されるため)。

## 5. プロンプト組み立ての変更

### 5.1 `src/server/prompts/presetParts.ts` — renderPresets

- 反復を `Object.entries(activePresets)` 任せにせず、**カテゴリ定義順の固定配列**(narration → aftertaste → emotionDisplay → sceneProgression → chapterEnding → painLevel → intimacy)で回す。
- 値が配列(aftertaste)の場合は、**項目ごとに独立ブロック**として出力する:

```
【読後感: 切ない】
届きそうで届かないもの、…

【読後感: ひりつく】
各場面に小さな不和・…
```

- 未知の id / 未知のカテゴリは従来どおりスキップ。全体見出し「【選択された設定】」と結合形式は現行を維持する。

### 5.2 `src/server/prompts/systemPrompt.ts` — 旧全文プロンプトの重複除去への対応

`normalizeAdditionalInstructions` は「旧UIが保存した基本プロンプト全文」から、現在の生成済みプロンプトと一致するブロックを除いて追加指示だけを残す仕組みだが、**プリセット text が全面的に変わるため、旧 text のブロックが一致せず追加指示側に漏れる**。

対策: 設定ブロックのフィルタに「旧カテゴリラベルの見出しを持つブロックは無条件で除去する」を追加する。対象ラベル(定数化):

```
ジャンル / 文体 / 視点スタイル / 語りの距離感 / 展開テンポ / 描写密度 /
会話量 / 関係性の進展速度 / 濡れ場の描写 / 禁止事項
```

`【<ラベル>: ...】` 形式で始まる設定ブロックが対象。「濡れ場の描写」は新カテゴリにも同名ラベルがあるが、新規生成分は生成済みプロンプト側との一致で除去されるため、このフィルタに含めて問題ない(旧全文内でユーザーが手改変していた場合は失われるが、許容する)。

## 7. UI 変更

### 7.1 共通コンポーネント `src/client/components/PresetSelector.tsx`(新規)

ProjectForm と WorkSettingsTab の両方で使う選択 UI。要件:

- 3グループに分けて見出しを付ける:
  - **境界設定**(読みたくないものを読まされない契約): painLevel, intimacy
  - **語りと構成**: narration, sceneProgression, chapterEnding
  - **読み味**: aftertaste, emotionDisplay
- 各項目は **label(太字)+ text 全文** を併記する(現行 WorkSettingsTab の intimacy 欄と同じ見せ方)。ラベルだけの表示は不可。
- narration 以外の単一選択カテゴリには「**指定しない**」選択肢を先頭に置く(選ぶとキーを削除)。
- aftertaste はチェックボックス。2件選択中は未選択項目を disabled にする。
- intimacy と painLevel には現行の説明文スタイルで補足を付ける:
  - intimacy: 「性的な場面をどう扱うかを選びます。指定しない場合、プロンプトには何も追加されません。」
  - painLevel: 「登場人物にどこまで辛いことが起きてよいかを選びます。安心して読みたいか、容赦ない展開を望むかの契約です。」

### 7.2 `src/client/components/ProjectForm.tsx`

- プリセット選択部を PresetSelector に置き換える。
- handleSubmit のハードコードされたフォールバック(`'modern-drama'` 等)を削除し、`{ ...DEFAULT_ACTIVE_PRESET_IDS, ...selection }` で構成する。
- 初期選択は narration=third-close のみ(intimacy の事前選択は無くなる)。

### 7.3 `src/client/components/WorkSettingsTab.tsx`
