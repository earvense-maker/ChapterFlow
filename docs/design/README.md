# 設計資料アーカイブ

このディレクトリには、ChapterFlow の実装時に使用した設計書・レビュー指示を履歴資料として保存する。

これらは作成時点の判断や移行理由を確認するための資料であり、現行仕様の正本ではない。現在の挙動はコード、[README](../../README.md)、[利用ガイド](../利用ガイド.md)、[リリースノート](../releases/)を優先する。

## 運用方針

- 実装完了後の設計書とレビュー指示は `archive/` に置く。
- 現在も反復して使う手順書はアーカイブしない。
- 内容が古いだけでは削除せず、意思決定の経緯として残す。
- 完全な重複、誤って生成された一時ファイル、別資料に全内容が統合済みと確認できたものだけ削除する。
- 新しい設計資料には作成日、対象バージョン、実装後の扱いを記載する。

現在も使う手順書: [受け入れテスト手順書](../../受け入れテスト手順書.md)
